# AI Job Recommendation System

A capstone project combining **Programming for AI (CS3703)** and **Machine
Learning**: a CV-based job recommender that classifies the most suitable job
roles for a candidate and estimates an expected salary, trained in Google
Colab and deployed as a Tkinter desktop application.

## Features

- **CV upload** — PDF, DOCX, or TXT, parsed automatically for skills and years of experience
- **Job role classification** — top 3 ranked roles with model confidence
- **Salary estimation** — regression model trained on the same dataset
- **ATS match score** — for each recommended role, an explainable score:
  `(matched required skills ÷ total required skills for that role) × 100`.
  Required-skill profiles per role live in `src/ats.py` and are reused at
  inference time — no retraining needed.
- **Personalized fit description** — a short written explanation per role
  that references your actual detected skills, years of experience,
  project count, and education, plus the specific skills you're missing
  for that role (generated by `src/ats.py`, not a generic template)
- **Dashboard-style desktop GUI** — sidebar + card-based results with
  color-coded ATS score badges (teal = strong match, amber = moderate,
  red = weak), built in Tkinter

## Dataset

[AI-Powered Resume Screening Dataset (2025)](https://www.kaggle.com/datasets/mdtalhask/ai-powered-resume-screening-dataset-2025)
on Kaggle — a plain public dataset, containing skills, experience, education,
job role, and salary expectation for 1,000+ synthetic resumes.

You've already downloaded this CSV yourself, so the notebook **does not
connect to Kaggle at all** — it simply prompts you to upload the file
through Colab's file picker, and accepts either the **`.zip`** exactly as
downloaded from Kaggle or the **`.csv`** if you've already unzipped it. No
Kaggle account or API token is required.
(If you'd rather automate the download in the future, `src/download_data.py`
is kept as an optional Kaggle-API-token-based downloader, but it isn't used
by the default notebook flow.)

## How to run this project

### Step 1 — Train the models in Google Colab

1. Upload this whole project folder to Google Drive (or clone it into Colab).
2. Open `notebooks/job_recommender_training.ipynb` in Colab.
3. Run all cells top to bottom. In **Section 2 ("Upload the dataset")**
   you'll be prompted with a file picker — select either the `.zip` file
   exactly as downloaded from Kaggle, or the `.csv` if you've already
   unzipped it. The notebook detects which one you gave it and extracts
   the CSV automatically if needed. This will:
   - clean the dataset
   - train a from-scratch NumPy softmax classifier (baseline)
   - train and tune the scikit-learn RandomForest classifier + regressor
   - log every run to MLflow (`mlruns/`)
   - save confusion matrices, MAE/RMSE plots, and metrics to `results/`
   - **save the final trained models** to `models/`:
     - `job_role_classifier.joblib`
     - `salary_regressor.joblib`
     - `skill_vectorizer.joblib`
     - `label_encoders.joblib`
4. Download the `models/` folder (the last notebook cell zips and downloads
   it automatically) and place it inside this project folder on your machine.

**The models only need to be trained once.** The notebook does not need to
be re-run unless you want to retrain on new data.

### Step 2 — Run the desktop app in VS Code

```bash
pip install -r requirements.txt
python app/main_app.py
```

The app loads the saved models from `models/` (no training happens here),
lets you upload a CV (PDF/DOCX/TXT), and displays:
- extracted skills and detected years of experience
- top 3 recommended job roles with confidence scores
- an estimated salary for the best-matched role

## Reproducibility

All random seeds, hyperparameters, and file paths live in `config.json` so
results are reproducible across runs and machines, per the course's
emphasis on reproducible AI systems (Week 3 / CLO2).

## Notes on architecture decisions

- **`src/` is shared** between the Colab notebook and the Tkinter app. The
  exact same cleaning/feature-extraction code is used for training and
  inference, avoiding train/serve skew.
- **Program logic is separated from interface code**: `app/main_app.py`
  contains only UI code; all CV parsing, feature building, and prediction
  logic lives in `src/recommend.py`, which the app imports.
- **From-scratch vs. library comparison**: the notebook implements a
  multinomial logistic regression with NumPy and gradient descent, then
  compares its accuracy against scikit-learn's `RandomForestClassifier`
  and `LogisticRegression`, matching the course's W10/W11 pedagogy.
